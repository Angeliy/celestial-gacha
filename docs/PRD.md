# Celestial Gacha PRD

## 1. 项目背景

本项目是一个国风水墨主题的网页抽卡模拟器，灵感来自 Stitch 设计稿 `Oriental Ink Gacha Simulator`。设计目标是将传统水墨、朱砂印章、金粉光晕和现代玻璃拟态结合，形成具有仪式感的抽卡体验。

## 2. 项目目标

- 实现一个可运行、可访问、可测试的 React 抽卡模拟器。
- 完成 4 个核心页面：祈愿池首页、抽卡页面、统计页面、角色图鉴。
- 实现 SSR 1.6% 概率和 90 抽保底逻辑。
- 给主要交互元素添加 `data-testid`，方便自动化测试定位。
- 输出自动化测试报告，支持后续手动验收。

## 3. 用户角色

目标用户是希望体验抽卡流程、验证概率规则、查看收集结果的玩家或课程作业评审者。

## 4. 功能范围

### 4.1 祈愿池首页

- 展示两个限定祈愿池 Banner。
- 展示 SSR 概率、保底次数、角色数量。
- 支持从 Banner 直接进行十连抽。
- 支持进入角色图鉴查看详情。

### 4.2 抽卡页面

- 展示累计抽数。
- 展示距离 SSR 保底剩余次数。
- 展示 SSR 保底进度条。
- 支持单抽。
- 支持十连抽。
- 支持重置抽卡记录。
- 展示最近 10 条抽卡结果。

### 4.3 统计页面

- 展示总抽数、SSR 数量、实际 SSR 率、当前保底。
- 展示 SSR / SR / R 稀有度分布。
- 展示 SSR 抽取历史。

### 4.4 角色图鉴

- 展示所有角色卡片。
- 已抽到角色显示完整档案。
- 未抽到角色显示未显影状态。
- 展示收集进度。

## 5. 抽卡规则

- SSR 基础概率：1.6%。
- SSR 保底：第 90 抽必出 SSR。
- 抽到 SSR 后，保底计数重置为 0。
- 未抽到 SSR 时，保底计数加 1。
- SR 概率：16%。
- 其余为 R。

## 6. 视觉规范

设计稿来源：Stitch 项目 `Oriental Ink Gacha Simulator`。

核心视觉：

- 深紫色主背景。
- 水墨云纹背景层。
- 金粉粒子质感。
- 玻璃拟态面板。
- SSR 金色光晕。
- SR 紫色高亮。
- R 青蓝色高亮。
- 朱砂竖排印章。
- 底部移动端导航栏。

## 7. 非功能需求

- 首屏可直接使用，不做营销落地页。
- 移动端和桌面端都能访问。
- 构建命令必须通过。
- 抽卡核心逻辑必须有自动化测试。
- 交互元素必须具备稳定的 `data-testid`。

## 8. 验收标准

- 用户能打开首页并看到祈愿池 Banner。
- 用户点击十连后能看到 10 条结果。
- 统计页能同步抽卡数据。
- 图鉴页能同步收集进度。
- 90 抽未出 SSR 时，第 90 抽必出 SSR。
- `npm run test` 通过。
- `npm run build` 通过。
